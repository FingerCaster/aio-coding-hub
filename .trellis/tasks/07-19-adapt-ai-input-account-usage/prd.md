# 兼容账户额度多种查询方式

## Goal

让账户用量展示根据供应商真实支持的查询契约返回余额或周期额度，兼容 NewAPI 的模型令牌计费与用户账户查询两种方式，并兼容 sub2api `rate_limits` 响应；任何无法证明语义的值都不得显示为余额。

## Background

- 现有 NewAPI 模型令牌 billing 流程会请求公开状态、subscription 和 usage 三个端点；有限额度令牌能够据此得到正确的总额、已用和剩余。生产入口当前固定选择该路径（`src-tauri/src/commands/providers/account_usage.rs:89-92`），解析公式位于 `src-tauri/src/domain/provider_account_usage.rs:685-786`。
- NewAPI 在启用令牌级统计且模型令牌为无限额度时，会在 subscription 中返回官方源码定义的固定无限额度哨兵。该值描述模型令牌策略，不是用户钱包余额，继续套用 billing 公式会产生伪余额。
- NewAPI 控制台的账户数据来自用户账户接口：余额和历史消耗分别由账户额度字段与累计消耗字段换算而来。该接口要求独立的系统访问令牌和匹配的数字用户 ID，模型 API Key 不能替代二者。现有配置虽然读取 `newApiUserId`（`src-tauri/src/domain/provider_account_usage.rs:175-184`），生产 billing 请求并未使用它。
- 一类 sub2api 成功响应只提供账户有效状态及 `rate_limits` 日周期额度，不提供根级 `balance`、`remaining` 或订阅对象；当前解析器只读取根级和 subscription 字段（`src-tauri/src/domain/provider_account_usage.rs:600-667`），因此只能显示“可用”。
- 共享结果 DTO 已有日、周、月字段（`src-tauri/src/domain/provider_account_usage.rs:66-71`），现有组件也会展示这些字段（`src/components/providers/ProviderAccountUsageInline.tsx:112-115`）；当前现场证据只确认 `window: "1d"`，不能据此推断周或月窗口编码。
- 两类改动共用账户用量 IPC、结果 DTO、展示组件和安全边界，作为一个任务按协议步骤实现和验收。

## Requirements

### R1. 保持 NewAPI 模型令牌 billing 兼容

- 有限额度模型令牌继续使用现有同源、禁止重定向、受限响应体的 status + subscription + usage 查询。
- 现有 USD 换算、负余额、过期状态和应用错误失败关闭行为不得回归。
- 官方固定的模型令牌无限额度哨兵必须映射为“可用”的无限额度状态，明确显示模型令牌无限且不返回 total、used 或 balance；不得把它解释为用户账户总额或余额，也不得用任意“大数阈值”猜测无限额度。

### R2. 支持 NewAPI 用户账户查询

- NewAPI 配置界面必须提供显式查询方式选择：“模型令牌额度”和“用户账户余额”；现有配置未声明查询方式时一律按“模型令牌额度”处理，保持向后兼容。
- 只有用户明确选择“用户账户余额”时，才要求并使用系统访问令牌和数字用户 ID；不得仅因历史配置中残留 `newApiUserId` 自动切换查询方式。
- 账户模式允许在系统访问令牌或 User ID 不完整时保存；编辑器和账户用量展示必须明确标记“需配置账户凭据”，不得发起部分认证请求，也不得影响供应商本身的保存、启用或路由。
- NewAPI 适配器应支持单独配置用户账户查询所需的系统访问令牌和数字用户 ID；不得把模型 API Key 当作用户访问令牌。
- 用户账户请求必须限制为规范化 Base URL 的同源 `/api/user/self`，禁止重定向，限制响应体，并分别发送 Bearer 系统访问令牌与 `New-Api-User` 请求头。
- 查询必须同时读取公开状态响应中的展示单位和换算基数，再验证账户响应的成功标志及有限数值字段；只实现证据支持的 USD 语义。
- 账户额度允许为负数；历史消耗必须为有限非负数。成功后映射为余额、历史已用和 USD 单位，不伪造总额度。
- 凭据缺失、ID 不匹配、认证失败、未知单位、字段缺失、非法数值、重定向、超限或无效 JSON 均失败关闭，不回退到伪造的 billing 余额。

### R3. 支持 sub2api 周期额度

- 当成功响应包含结构化 `rate_limits` 时，先支持证据明确的 `window: "1d"`，解析其 `limit`、`used`、`remaining`、窗口起点和重置时间。
- 已用与总额必须满足有限、非负及内部一致性校验；不能仅凭相似字段名猜测账户余额。
- `1d` 应映射到共享结果 DTO 已有的日周期指标并由现有账户用量组件展示；未知周期不得猜测为日、周或月指标。
- 仅有账户有效状态且没有可识别周期时，保持现有“可用”兼容行为；畸形的已知周期必须失败关闭，不能静默显示部分错误额度。

### R4. 选择、隔离与安全

- 查询方式必须由可解释的配置和响应契约决定，不能通过供应商名称、域名或实际账户数值硬编码。
- 系统访问令牌不得存入会返回前端并被供应商分享原样导出的 provider extension `values_json`（`src-tauri/src/domain/providers/queries.rs:149-195`、`src-tauri/src/domain/providers/share.rs:851-904`）；前端可读的 provider summary 只允许暴露“已配置”状态，不得回显令牌。
- 内置账户用量扩展的所有写入、迁移、完整导出导入及单供应商分享路径必须共用规范化边界，从扩展 JSON 中移除历史 User ID 和任何账户令牌字段，防止绕过私有凭据存储。
- 系统访问令牌如经保存 IPC 传入，前端调用日志必须像现有 API Key 一样在调用前脱敏（`src/services/providers/providers.ts:258-274`）；任何 IPC 结果、错误、普通日志或生成文件都不得包含真实值。
- 用户主动执行的完整配置导出必须包含系统访问令牌，完整配置导入必须原样恢复该令牌及查询模式；该行为沿用现有 API Key 与 OAuth 凭据的敏感配置备份语义和界面提示（`src-tauri/src/infra/config_migrate/mod.rs:65-111`）。
- 完整配置格式升级不得改变既有 v1 旧格式和 v2 完整 Skill 载荷的导入语义；版本能力判断必须按各特性首次出现的 schema version 分支，不能把“当前版本”常量同时当作所有特性的最低版本。
- 单供应商分享必须保留显式“用户账户余额”模式，但不得携带系统访问令牌或 User ID；导入后应处于明确的“需配置账户凭据”状态，不得回退到模型令牌 billing 或沿用分享者账户身份。
- 本机“复制供应商”必须复制系统访问令牌、User ID 和显式查询模式，使副本立即保持与来源供应商相同的账户查询行为；该本地复制不得把凭据写入日志或前端摘要。
- 切换 NewAPI 查询方式或切换/关闭账户用量适配器时必须保留已保存的系统访问令牌和 User ID，但非账户路径不得读取或发送它们；界面应提供独立、明确的“清除账户凭据”操作，只有该操作才删除整组账户凭据。
- 真实凭据、数字用户 ID、真实主机、完整响应、个人字段及实际账户金额只允许在当前会话内用于只读现场验收；不得持久化到仓库或任务材料。
- 测试必须使用合成凭据、保留测试主机和合成金额。
- 账户用量继续仅用于展示，不得改变路由、熔断、供应商启停/排序、可用性或 OAuth 配额。

## Acceptance Criteria

- [ ] 有限额度 NewAPI 模型令牌仍通过三端点 billing 契约显示正确总额、已用和余额。
- [ ] 旧 NewAPI 配置和显式“模型令牌额度”配置继续走 billing 契约，不因历史 User ID 残留改变行为。
- [ ] 显式“用户账户余额”配置在凭据完整时走用户账户契约并显示余额与历史消耗，配置不完整时给出明确的配置缺失状态。
- [ ] 账户模式在凭据不完整时仍可保存，且不会发送账户请求、回退到 billing 伪余额或改变供应商路由行为。
- [ ] NewAPI 无限模型令牌显示明确的无限额度可用状态，total、used、balance 均为空，不显示哨兵推导出的伪余额。
- [ ] 用户账户请求的 URL、重定向、请求头、响应上限、应用错误顺序、单位和数值语义均有合成测试覆盖，且任何失败都不返回部分金额。
- [ ] sub2api `rate_limits` 的 `1d` 窗口能显示日额度；当前无根级余额的成功响应显示日额度而不伪装成账户余额。
- [ ] 未知周期、畸形窗口、负数、非有限值及内部不一致响应按约定失败或忽略，且保留原有 sub2api 根级余额/套餐解析行为。
- [ ] 前后端类型、配置保存、查询结果和展示测试覆盖两种 NewAPI 路径及 sub2api 周期路径。
- [ ] 完整配置导出/导入能够往返恢复合成系统访问令牌和显式查询模式，v1/v2 导入行为保持不变，且导入导出日志、错误及测试快照不泄露令牌。
- [ ] 单供应商分享内容不包含合成系统访问令牌或 User ID；导入账户模式供应商后明确要求配置接收方自己的账户凭据。
- [ ] 本机复制账户模式供应商后，副本保留合成系统访问令牌、User ID 和查询模式，并能独立完成账户查询；复制过程无凭据回显或日志泄露。
- [ ] 查询方式及账户用量适配器来回切换不会自动删除或误用账户凭据；显式清除后令牌和 User ID 均被删除，账户模式显示“需配置”，模型令牌模式仍正常查询。
- [ ] 差异审计确认没有真实凭据、用户 ID、主机、响应体、个人字段或实际账户金额进入任何持久化文件。
- [ ] 账户查询不会触发或修改供应商路由、熔断、可用性、启停、排序或其他额度系统。

## Out of Scope

- 不自动登录供应商控制台，不保存浏览器 Cookie，不抓取网页 DOM。
- 不把账户用量结果用于流量调度或健康判断。
- 不为未知供应商设计通用 JSON 猜测器。
- 不在本任务中处理旧 Trellis 父子任务的状态或归档。
